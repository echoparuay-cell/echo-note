# บรีฟ Cowork: ตารางหมายเลขจอง 50 เบอร์ (เชื่อม Google Sheets)
## โครงการ Gratitude Guitars Phuket Model — phuket.skecho.org

ไฟล์หน้าเว็บล่าสุด: `phuket-gratitude-guitars-v6.html`
งานคือ **แทนที่ตารางสถานะหมายเลขแบบนิ่ง (3 แถว)** ด้วยกริด 50 เบอร์ที่ดึงสถานะจริงจาก Google Sheets

---

## 1. โครงสร้าง Google Sheets

สร้างชีตใหม่ชื่อ `Phuket-Booking` คอลัมน์ดังนี้:

| คอลัมน์ | ตัวอย่าง | หมายเหตุ |
|---|---|---|
| A: number | PH001 – PH050 | สร้างครบ 50 แถวตั้งแต่ต้น |
| B: status | available / pending / booked | ค่าเริ่มต้น available ทุกแถว |
| C: round | 1–5 | รอบละ 10 เบอร์ (PH001–010 = รอบ 1 ฯลฯ) |
| D: current_round | 1 | ใส่เฉพาะเซลล์ D1 — รอบที่เปิดอยู่ตอนนี้ |
| E: initials | ชื่อย่อผู้จอง | อาจารย์กรอกเอง |
| F: date | วันที่มัดจำ/วันที่เริ่ม pending | ใช้ดูครบ 24 ชม. ด้วยตาเอง |
| G: note | บันทึกอื่นๆ | ไม่แสดงบนเว็บ |

**สิ่งที่ส่งขึ้นเว็บ: เฉพาะคอลัมน์ A, B, C และค่า D1 เท่านั้น** — ห้ามส่ง initials/date/note ออกสาธารณะ

## 2. Apps Script (ใช้ pattern เดิมของ EchoHub / echoparuay-cell.github.io)

- `doGet()` อ่านชีต → คืน JSON: `{ current_round: 1, numbers: [{n:"PH001", s:"available", r:1}, ...] }`
- Deploy เป็น Web App (Anyone with the link) เหมือนที่ทำกับระบบสมาชิก EchoHub
- ใส่ header `Cache-Control` สั้นๆ หรือให้ frontend เติม `?t=timestamp` กัน cache

## 3. Frontend (แก้ใน v6)

- แทนที่ `<section>` "สถานะหมายเลข" เดิม ด้วยกริดปุ่ม 50 ช่อง (5 คอลัมน์บนมือถือ)
- โหลดหน้าแล้ว fetch JSON หนึ่งครั้ง + ปุ่ม "รีเฟรชสถานะ" ให้กดเช็กซ้ำได้
- สีตามธีมน้ำเงินของหน้า:
  - 🟢 available → พื้นเขียวอ่อน กดแล้วเปิด LINE พร้อมข้อความ `สนใจจองหมายเลข PHxxx`
    (ลิงก์: `https://line.me/R/oaMessage/@musicsupport/?สนใจจองหมายเลข%20PHxxx`)
  - 🟡 pending → เหลือง ข้อความ "รอยืนยันมัดจำ" กดไม่ได้
  - 🔵 booked → น้ำเงินเข้ม (--blue-deep) "จองแล้ว" กดไม่ได้
  - ⚪ นอกรอบปัจจุบัน (r > current_round) → เทา "ยังไม่เปิดจอง"
- แสดงบรรทัดสรุปเหนือกริด: "รอบที่ X — เหลือ Y หมายเลข"
- ถ้า fetch ล้มเหลว: แสดงข้อความ "เช็กสถานะล่าสุดได้ทาง LINE @musicsupport" แทน (ห้ามโชว์กริดค่าเก่าผิดๆ)

## 4. กติกาการดำเนินงาน (ฝั่งอาจารย์ — ไม่ต้องเขียนโค้ด แค่รับทราบ)

- ลูกค้าทัก LINE เลือกเบอร์ → เปลี่ยน status เป็น pending ทันที + ลงวันที่
- มัดจำเข้า → เปลี่ยนเป็น booked
- pending ครบ 24 ชม. ไม่โอน → เปลี่ยนกลับ available
- ชนกัน: ยึดเวลาข้อความ LINE ใครทักก่อนได้ก่อน

## 5. เช็กลิสต์ก่อนปล่อย

- [ ] ทดสอบเปลี่ยนสถานะในชีต → รีเฟรชหน้าเว็บแล้วสีเปลี่ยนตาม
- [ ] ทดสอบกดเบอร์ว่างบน iPhone → เปิด LINE พร้อมข้อความหมายเลขถูกต้อง
- [ ] ทดสอบกรณีปิด Apps Script → หน้าเว็บแสดงข้อความ fallback ไม่พัง
- [ ] ตรวจว่า JSON ไม่มีข้อมูลส่วนตัวผู้จองหลุดออกมา
- [ ] อัปเดตขั้นตอนการจองข้อ 2 ในหน้าเว็บให้สอดคล้อง ("ตรวจสอบสถานะหมายเลข" → กดจองจากกริดได้เลย)
